## 1. 作った背景

私は週４〜５日ほどランニングをします。
短くて30分ほど、週末には120分くらい走ることもあります。

特に長い距離を走る時ほど、音楽を聴きながら走ることが多いです。

しかしながら、毎回同じプレイリストを流しているととっても飽きるのです。そして、シーン（朝公園を走る、夜川沿いを走る、今日はスピードを上げたいなどなど）に適した音楽を聴きたいわけです。

ただプレイリストを組むって意外と大変だし（特に選曲が）、アルバムやミュージックアプリレコメンドのプレイリストを選ぶのもなんか違う。

そんな時、プレイリストを自動で組んでくれるアプリがあったらいいなとか思ったりしてました。そんなただただ自分が欲しいと思ったアプリケーションをドッグフーディング的な形で作り磨いていこうと思ったのです（ワンだけにね🐕）


## 2. RunTuneとは何？

改めまして今回作ったのは、ランニングの条件（時間・BPM・ジャンル・気分・好きなアーティスト）を入力すると、AIが最適なプレイリストをリアルタイムで生成してくれるモバイルファーストのWebアプリです。

### 主な機能

| 機能 | 概要 |
|------|------|
| AIプレイリスト生成 | 入力条件をもとに gpt-4o-mini が曲数・BPM・多様性を考慮して選曲 |
| 生成結果の確認 | 生成されたプレイリストの詳細を確認 |
| プレイリスト保存 | 気に入ったプレイリストに名前をつけてデバイスに永続保存 |
| 保存一覧・削除 | 保存済みプレイリストの閲覧・曲リスト展開・削除が可能 |
| 再生成 | 同じ条件でワンタップ再生成 |

### 技術スタック
**フロントエンド**: Next.js (App Router) + TypeScript strict mode
**スタイリング**: Tailwind CSS + カスタム CSS アニメーション
**バックエンド**: Supabase /  localStorage
**AI**: OpenAI API (gpt-4o-mini)
**デプロイ**: Vercel

## 5. 苦戦したポイント

### プレイリスト提案の精度向上

初期実装では生成される曲数がばらついたり、同じアーティストに偏るケースがあったため、より精度の高い提案となるようプロンプトを繰り返し調整しました。

- 目標曲数をランニング時間から逆算して明示的に指定
- 合計再生時間の許容誤差を ±90 秒に制限
- 同一アーティストの重複を最大 2 曲までに制約
- レスポンスのトークン上限を引き上げ、曲数が多い場合の出力切れを防止

などなど、なるべく自分の趣味に近い音楽が選ばれてほしい。一方で普段聴くことがなかった新たな音楽も織り込んでほしい。この塩梅の調整が今なお課題となっています。

### Apple Musicに連携したい
選曲する手間は省けましたが、使用する音楽アプリのなかでプレイリストを作成する際は手動で対応する必要があります。
なので気に入ったプレイリストはワンタップで自身のApple Musicにも追加されるようにしたかったのですが、2つの大きな壁が立ち塞がり、このタイミングでは実装は見送ることにしました。
- 費用の壁
- 技術的な難易度の壁（主にセキュリティ）

一旦は諦める方向

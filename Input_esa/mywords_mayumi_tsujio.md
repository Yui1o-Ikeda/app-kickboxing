他の英語学習アプリとの一番の違いは、
### 教材が「自分の言葉」でできていること

**教科書の例文ではなく、今日ランチで食べたものや、
週末の旅行の話が、そのまま英語の教材**になります。
だから記憶に残りやすい。


## きっかけは「続かない」という課題感
英語学習アプリはたくさんあります。
スピフルも使ったし、SPEAKなども使いました。
オンライン英会話も続けていました。

でもあんまり楽しくない、続かない、

## 理由のひとつが「自分ごとになりにくい」こと

誰かが作った例文を繰り返すしても、
「ジョンが買い物に行きました」とか、誰の話なんや？となりませんか。

**自分が話したいことを話す方が、モチベーションが続くはずだ**と思いました。
別にすごいレベル高い英語が話したいわけではなくて
コミュニケーションを取れるレベルだったらOKだと思っています。

- 今日何があったか
- 何が楽しかったとか
- 好きなことについて語りたい！とか

こんな日常のことを話せるようになりたい！
だからハードルも意識しました。

**まずは日本語で話すだけ、10〜30秒で完結**。
それだけで英語の教材が生まれる体験だったら簡単じゃないですか？


## Claude Codeとの開発体験

今回、開発にはAnthropicの Claude Code を使いました。
コードを書いてもらうだけでなく、アイデアの壁打ちから一緒に始めました。

### ① アイデアの壁打ちと要件設計
最初にやったのは、頭の中にあるぼんやりしたアイデアをClaudeに投げることです。

「こんな気持ちで作りたい」「ゲーミフィケーションの要素が欲しい」「続けたくなる仕掛けが必要」——そういった感覚の話を伝えていくと、Claudeが機能として整理してくれました。連続記録（ストリーク）、バッジ獲得、単語図鑑のコレクション要素……これらはすべて、壁打ちの中から生まれたアイデアです。

### ② 仕様書（MDファイル）にまとめる
アイデアが固まったところで「これを仕様書としてMDファイルにまとめて」とお願いしました。
画面構成、機能要件、技術スタックの候補が1つのファイルに整理されます。これが開発の地図になりました。

**作ってもらったファイルは下記**

- 06_TECH_STACK.md
- 05_DATA_STRUCTURE.md
- 04_SCREENS.md
- 03_FEATURES.md
- 02_USER_FLOW.md
- 01_OVERVIEW.md
- README.md


### ③ MDファイルを渡して開発スタート
「このMDファイルを元に開発してください」

そこからはClaudeがコードを書いていきます。
技術スタックもClaudeと相談しながら決めました。
今回はNext.js + TypeScript + Zustandの構成です。

利用したのはこちらですがReactなど使わなくても良かったかも？そこまで高度なことをしていないので
| 技術 | 役割 |
|------|-----------|
| Next.js | メインフレームワーク（画面 + API） |
| React  UI構築のライブラリ |
| TypeScript | 安全なJavaScript |
| Google Gemini AI | 翻訳・AI機能 |
| uuid | ユニークIDの生成 |

### 苦労したこと——APIの壁
一番詰まったのは、Gemini APIの無料枠があっという間に尽きたことです。
そんなに大したことはしていないのですが

このアプリでは、日本語のテキストを英語に翻訳し、
さらに重要単語を抽出・解説するためにGoogle GeminiのAPIを使っています。

開発中にAPIを何度も叩いていると**あっという間に無料枠がなくなり、エラーが出るようになりました。**

解決策はシンプルで、初めてAPIに課金しました。

クレジットカードを登録して、有料プランに切り替える。
それだけのことなのですが、**「APIにお金を払う」という経験が初めてだったので、少し緊張**しました。


## 発見—AIのAPI・デプロイなど初体験のハードルが低い

翻訳も、単語抽出も、発音記号の付与も、数行のコードとAPIキーで動きます。「AIを使ったサービスを作る」というのは、以前は大きなハードルに見えていましたが、今は「適切なAPIを選んで、呼び出し方を覚えるだけ」という感覚になりました。

ただし、AIを使いすぎるとあっという間に課金させそうというドキドキは伴いました


## まとめ—

コードが書けなくても、アイデアを言語化できれば、Claudeが設計を手伝い、コードを書いてくれます。詰まったときも一緒に原因を探してくれます。開発体験というより、共同制作に近い感覚でした。

My Wordsはまだプロトタイプですが、「自分の言葉で英語を学ぶ」という体験は、
すでにちゃんと動いています。ぜひ触ってみてください。

社会人11年間ビジネス職一筋、半年前まではChatGPTをGoogle検索代わりに使ってたAIリテラシー低め人間でも、Claude Codeを使えば簡単にWebアプリを作ることができるのか、のレポートです😌

⚠️技術的なことはほぼわかってないので技術的な正確性より「AIリテラシーの低い人間でもClaude Codeを使えばWebアプリ開発できるんだ」という雰囲気で読んでいただけると幸いです😂

## 何を作ったか

**KICKLOG** — キックボクシングのトレーニング記録と費用対効果を管理するスマホ向けWebアプリです。
<https://app-kickboxing.vercel.app/>

| 機能 | 概要 |
| --- | --- |
| カレンダー | 通った日（実績）と予定日を月ごとに表示 |
| セッション記録 | ラウンド数・種類・追込み度・トレーナー・コメントを記録 |
| ROIダッシュボード | 月会費＋交通費から「1回単価」「1ラウンド単価」をリアルタイム計算 |
| 目標進捗 | 月の目標回数・目標ラウンドに対する進捗をバー表示 |
| クロスデバイス同期 | スマホ・PCどちらからでも記録を確認できる |

## 作った背景

実は3年間ほどキックボクシングジムに通ってます！運動不足の解消＋人と接しながら運動したい＋今までやったことのないスポーツをやってみたい、というのが動機です。おかげで体力もつきましたし、ハイキックもできるようになりました。（でもスパーリングとかはしないので対人で使えるかは謎です、というか使うシーンが来なくていいですw）

一方で、通い始めてからずっと気になっていたことがあって、
**月会費¥17,600払ってるけど、本当に元が取れてるのか問題です。**

感覚はあるものの数字で把握できていないのが地味にモヤっていました。通った回数はGoogleカレンダーに登録していたので見返せばわかるんですが、1回あたり・1Rあたりいくらになるかとか、交通費込みのトータルコストに対してどれぐらいなのかをちゃんと計算したいなとずっと思ってました。
（毎月合計約2万程度の出費なので、幾度となく経費削減してNISAとかに回そうかな〜と考えてしまうので。。笑）

今回の施策が発表されたとき、作るものの基準として「習慣的に使うもの」にしようと決めていたので即決でした！
（正直なところ、自分が本当に課題感じてるやつじゃないと作るモチベーションも改善し続けるモチベーションにも繋がらないのが本音ですw）

## 開発の流れ

基本的には日本語でClaude Codeに話しかけるだけです。

1. 「こんなアプリ作りたい」と要件を伝える
2. 実装してもらう
3. 動かして確認する
4. 気になるところを「ここをこう変えて」と伝える
5. 繰り返す

コードは一行も書いていないです！
エラーが出たり上手くいかないシーンがあれば、エラーメッセージをそのままコピペ or スクショを貼り付ければ、原因を調べて修正してくれます。

（「こういう画面にして」「この色を変えて」みたいな感じで進められるので、思っていたよりハードルは全然高くなかったです😌）

## 苦戦したポイント

### データ保存の永続化

最初はブラウザ（localStorage）にデータを保存する設計で動かしていたんですが、これだと**スマホから見ると記録が全部消えている**という状態になってしまいまして。笑

「端末をまたいでデータ保持したい」と伝えたところ、ClaudeがSupabaseというクラウドデータベースへの移行を提案してくれて実装へ。

ただここからがなかなかでした。移行後にこんなエラーが出まして。

```text
Could not find the table 'public.sessions' in the schema cache
```

原因はテーブルのアクセス権限の設定が足りていなかったことで（そういう概念があることすら知らなかったですw）、Supabaseのダッシュボードで設定を変更して解決しました。

エラーの意味は全然わからなかったんですが、エラーテキストをそのままClaudeに投げると「これが原因で、こうすれば直ります」と説明しながら解決策を出してくれるのがありがたかったです！

### エラーが「保存に失敗しました」しか表示されない問題

移行後に試しに記録を登録しようとしても「保存に失敗しました」としか表示されず、どこが悪いのかわからないという状況にも陥りました。
Claudeに相談したら「エラーの詳細を表示するようにコードを変えましょう」と提案してくれて、コードの書き換えもやってくれたので、エラーの詳細を把握できるようになり一気に解決が進みました。
（「何をどう聞けばいいか」のナビもしてくれるのが地味に助かりました）

## やってみた学びと所感

### よかった点

- 「作れるかどうか」を考えなくていい。「こうしたい」だけ考えられる
- エラーの調査・修正をClaudeが担ってくれるので、自分は動作確認に集中できる
- 毎回少しずつ進んだ状態で終われるので達成感がある！！

### 難しかった点

- 何を作るか・どういう体験にするか、の設計は自分でやる必要がある
  → これは改めてですが、要件やペイン・実現したいことというInputの重要性ですね
- 技術的なエラーの意味がわからないと「何を聞けばいいか」がわからないことがある
  → 考える時間がもったいないので、エラー文をそのまま貼る・キャプチャを貼るだけでClaudeが回答してくれます
- デプロイ周りの設定（VercelとGitHubの連携・環境変数の設定）は手順が多くて詰まりやすい
  → とはいえここも詰まったらClaudeに聞けばなんとかなりました！

### 感想

正直、ここまで簡単にできるとは思っていなかったです。（初回版をデプロイするまで15分くらい？そこから改善を重ねて今のバージョンにするまで1時間もかからなかったです）

ROIの計算ロジックも、カレンダーUIも、クラウドDBへの移行も、「こうしたい」と日本語で伝えるだけで実装が進んでいく感覚は最初ちょっと不思議でした。（すごい時代になった、というよりも、正直少し恐怖を感じましたw）

ただ、**「何を作るか」「どういう体験にするか」は自分で決め続ける必要があります。** ClaudeはあくまでWebアプリ作りの相棒であって、アプリの目的や使い心地の判断は自分のままです。コードが書けないことが障壁にならなくなった分、「作る理由・解きたい課題を言語化する力」の重要性をすごく感じました。

日々の仕事や生活の中で「こういうのあったら便利かな？」「AIで自動化できないかな？」を考えて、アイデアベースでClaudeと壁打ちして実装する！というのをどれだけ試せるか・スプリントを回せるかが、結果的にAIリテラシーを高めていくことに繋がりそうですね💪
